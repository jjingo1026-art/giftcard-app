📖 이 앱은 어떻게 구성되어 있나요?

이 앱은 크게 두 부분으로 나뉩니다.
① 백엔드(서버): 고객·관리자·담당자의 요청을 처리하고 DB에 저장합니다.
② 프론트엔드(화면): 사용자가 실제로 보는 웹 페이지입니다.

채팅은 Socket.IO라는 기술로 실시간 통신하고, 외국어 메시지는 Google Translate API로 자동 번역합니다.

💡Rate Limit이란? 한 IP에서 짧은 시간에 너무 많은 요청을 보내면 차단합니다. 스팸이나 해킹 시도를 방어합니다.
// 로그인 시도 제한: 15분 동안 최대 20번만 허용

  windowMs: 15 * 60 * 1000,  // 15분 (밀리초 단위)
  max: 20,                    // 최대 20번
  message: { error: "요청이 너무 많습니다." }
});

// 채팅 읽기: 1분에 최대 60번 허용
const chatReadLimiter = rateLimit({ windowMs: 60*1000, max: 60 });

📄 artifacts/api-server/src/lib/translate.ts
역할: 외국어 메시지를 한국어(및 기타 언어)로 번역합니다.
Google Translate를 먼저 시도하고, 실패하면 MyMemory API로 폴백(대체)합니다.
💡폴백(Fallback)이란? 첫 번째 방법이 실패했을 때 대안을 사용하는 패턴입니다. Google → 실패 → MyMemory로 자동 전환합니다.
// 지원 언어 목록: 영어, 중국어(간체/번체), 베트남어, 일본어, 태국어, 러시아어, 몽골어, 인도네시아어
export const SUPPORTED_LANGUAGES = ["en", "zh-CN", "zh-TW", "vi", "ja", "th", "ru", "mn", "id"];

// Google Translate 비공식 무료 엔드포인트
async function translateViaGoogle(text, from, to) {
  const url = `https://translate.googleapis.com/translate_a/single?client=gtx&sl=${from}&tl=${to}&dt=t&q=${text}`;
  const resp = await fetch(url, { signal: AbortSignal.timeout(6000) }); // 6초 안에 응답 없으면 포기
  // 응답 데이터에서 번역된 텍스트만 추출
  const translated = data?.[0]?.map(item => item?.[0]).filter(Boolean).join("");
  return translated || null;
}

// 1차(Google) 실패 시 2차(MyMemory) 시도
async function translateText(text, from, to) {
  const google = await translateViaGoogle(text, from, to);
  if (google && google !== text) return google; // Google 성공 → 바로 반환
  const mymemory = await translateViaMyMemory(text, from, to);
  if (mymemory) return mymemory;    // MyMemory 성공 → 반환
  return text;                        // 둘 다 실패 → 원문 반환
}

// 모든 지원 언어로 동시에 번역 (Promise.all = 병렬 처리)
export async function translateAll(text, sourceLang) {
  const result = {};
  await Promise.all(                // 모든 언어를 동시에 번역 (순서대로 하면 너무 느림)
    allLangs.map(async (lang) => {
      result[lang] = await translateText(text, sourceLang, lang);
    })
  );
  return result; // { ko: "안녕하세요", en: "Hello", zh-CN: "你好", ... }
}
📄 artifacts/api-server/src/lib/jwt.ts
역할: 로그인 인증 토큰(JWT)을 생성하고 검증합니다.
로그인 성공 시 토큰을 발급하고, 이후 모든 요청에서 이 토큰으로 "진짜 로그인된 사용자인지" 확인합니다.
ℹ️JWT(JSON Web Token)란? 로그인 후 서버가 발급하는 디지털 신분증입니다. 클라이언트가 이 토큰을 헤더에 담아 요청하면 서버가 신뢰합니다. Access Token(15분 유효) + Refresh Token(7일 유효) 두 가지를 사용합니다.
// Access Token: 15분 유효 (짧게 → 보안)
export function generateAccessToken(payload) {
  return jwt.sign(payload, process.env.JWT_ACCESS_SECRET, { expiresIn: "15m" });
}

// Refresh Token: 7일 유효 (길게 → 편의)
export function generateRefreshToken(payload) {
  return jwt.sign(payload, process.env.JWT_REFRESH_SECRET, { expiresIn: "7d" });
}

// 토큰 검증: 위조·만료 여부 확인
export function verifyAccessToken(token) {
  return jwt.verify(token, process.env.JWT_ACCESS_SECRET); // 실패 시 예외 발생
}

// 핵심 함수: 메시지를 현재 보는 언어로 변환
export function getTranslated(msg, viewLang) {
  const sourceLang = msg.language ?? "ko";
  
  // 원문 언어와 보는 언어가 같으면 그냥 원문 반환
  if (viewLang === sourceLang) return msg.message;
  
  // 번역된 텍스트에서 해당 언어를 찾아 반환
  // 없으면 원문 반환 (번역 전이거나 실패한 경우)
  return msg.translatedText?.[viewLang] ?? msg.message;
}

ℹ️ORM이란? SQL 쿼리를 직접 쓰지 않고 TypeScript 코드로 DB를 다루는 도구입니다. db.select().from(table).where(...) 형태로 사용합니다.
// 예약 테이블
reservationsTable: {
  id,           // 예약 번호 (자동 증가)
  name,         // 고객 이름
  phone,        // 전화번호
  quantity,     // 상품권 수량
  kind,         // "paper"(지류) 또는 "mobile"(모바일)
  status,       // "pending"(대기) / "confirmed"(확정) / "completed"(완료)
  pin,          // 4자리 인증 번호 (고객 확인용)
  staffId,      // 배정된 담당자 ID
  createdAt,    // 예약 신청 시각
}

// 채팅 테이블
chatsTable: {
  id,              // 메시지 번호
  reservationId,   // 어느 예약의 채팅인지
  sender,          // "customer" / "admin" / "staff" / "system"
  senderName,      // 표시 이름 ("고객", "관리자", "홍길동")
  message,         // 원문 메시지
  language,        // 원문 언어 코드 ("en", "vi", "ko" 등)
  translatedText,  // 번역 결과 { ko: "안녕", en: "Hello", vi: "Xin chào" }
  read,            // 읽음 여부 (true/false)
  time,            // 전송 시각
}

staffTable: {
